PICOBOT — FINAL REPO STRUCTURE
==============================

Top-level
---------

ARCHITECTURE.md
GUIDE.md
README.md
  Documentazione principale del progetto.

Makefile
  Entry point operativo per sviluppo locale, sandbox docker, bootstrap tools,
  test e avvio del sistema.

dev/
  Materiale operativo di supporto allo sviluppo.

dev/scripts/
  Script helper per asset di test, bootstrap locale e supporto operativo.

docker/
  Materiale Docker del sandbox runtime locale.

infra/
  Materiale infrastrutturale locale non applicativo.

infra/searxng/
  Configurazione del backend search locale.

docs/
  Documentazione tecnica del repository e dei flow.

picobot/
  Codice applicativo principale del progetto.

tests/
  Test automatici.

tests/fixtures/data/
  Dati e asset usati nei test.

Responsabilità principali
-------------------------

picobot/agent/
  Orchestrazione del turno, workflow dispatcher, decision execution.

picobot/app/
  Bootstrap e avvio dell'applicazione.

picobot/bus/
  Event bus e messaggi interni.

picobot/channels/
  Surface adapters (CLI, Telegram, ecc.).

picobot/config/
  Config schema, loader e init.

picobot/context/
  Layer di context building condiviso.
  Area candidata a diventare centrale nel prossimo step architetturale.

picobot/cron/
  Supporto a cron/service tick.

picobot/heartbeat/
  Heartbeat runtime.

picobot/knowledge/
  Materiale semantico e routing knowledge docs.

picobot/memory/
  Gestione facts, summary, memoria applicativa.

picobot/prompts/
  Prompt utilities e language helpers.

picobot/providers/
  Provider LLM e registry provider.

picobot/retrieval/
  Retrieval KB/RAG locale.

picobot/routing/
  Routing policy, retrieval router, ranking e decision.

picobot/runtime/
  Runtime event-driven e publisher eventi/output.

picobot/sandbox/
  Esecuzione su backend sandbox.

picobot/services/
  Servizi trasversali, incluso search backend.

picobot/session/
  Session state e persistenza sessione.

picobot/tools/
  Tool registry e implementazioni concrete (tts, stt, web, python, yt, ecc.).

picobot/ui/
  CLI rendering e command layer locale.

picobot/utils/
  Utility generiche.

Note
----

- Il codice applicativo vive in picobot/
- I dati di test vivono in tests/fixtures/data/
- Il runtime sandbox vive in docker/
- L'infrastruttura locale di supporto vive in infra/
- Gli script operativi vivono in dev/scripts/
